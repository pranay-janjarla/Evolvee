The problem

Most forms are frozen the day they ship.

A team writes the questions, deploys, and reviews results by hand weeks later. That gap quietly hides four expensive failures.

Survey fatigue

When 95% pick the same answer, the question carries zero signal — but it stays, because nobody noticed.

Silent skips

Shown 200 times, answered 80 — a 60% skip rate. You see the answers, never the signals of confusion.

Delayed signal

A safety spike takes days to surface because someone has to download and review the data by hand.

Domain lock-in

Most tools hardwire detection logic for one domain and can't carry it to a new context.

How it works

Three agents. One job each.

Not one overloaded prompt — a sequential pipeline where each agent hands off structured JSON, so none can hallucinate about what the last one found.

01

Analyst

"What patterns exist?"

Reads submissions and telemetry, flags anomalies and low-variance questions, classifies severity.

"Should the form change?"

"Write it to memory."

Architecture

Thin at the top, rich at the bottom.

Five layers, each with one bounded responsibility. Submissions flow down; findings, alerts, and learned context flow back up.

What makes it different

Built to learn, not just collect.

Deterministic confidence scoring

Auditable scores — not the model's mood — decide what auto-applies, what waits for review, and what gets discarded. Noisy data never mutates your form, and the same input always yields the same decision.

Question telemetry

Tracks engagement, not just answers — skip rates and distributions reveal confusion and saturation.

Living workspace context

A per-workspace memory that accumulates baselines and findings over the life of the deployment.

Hybrid triggers

Responds to activity, not the clock — scheduled runs, surge-based event triggers, and safety rechecks coexist.

Policy guardrails & human-in-the-loop

GDPR, HIPAA, anonymity, and question-count limits live in the agents' context — proposals self-censor before they reach you. Autonomous by default, review by exception: high-confidence changes apply instantly, uncertain ones queue for one click of approval.

Quickstart

Running in under a minute.

bash
# clone the framework
git clone https://github.com/pranay-janjarla/Evolvee.git
cd Evolvee

# install dependencies
pip install -r requirements.txt

# add your Anthropic API key
echo "CLAUDE_API_KEY=sk-ant-..." > .env

# launch the server + evolution loop
python server.py
  • No database setup. SQLite ships in the box — clone and run.
  • Forced structured output. Every Claude call returns valid JSON via tool use.
  • Prompt caching built in. The shared context is paid once, then reused ~10× cheaper.
  • Fully reversible. Form versions are append-only — roll back to any prior state.
